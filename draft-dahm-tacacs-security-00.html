TACACS+ TLS Support
draft-dahm-tacacs-security

Abstract

The TACACS+ Protocol provides device administration for routers, network access servers and other networked computing devices via one or more centralized servers. This document adds Transport Layer Security (TLS) to TACACS+.

1. Introduction

The TACACS+ Protocol provides device administration for routers, network access servers and other networked computing devices via one or more centralized servers. The protocol carries authentication, authorization and accounting information for TACACS+ clients. While this information is highly sensitive, TACACS+ lacks modern and suitable privacy, integrity and authentication of the connection and network traffic between the server and client. Its existing mechanism is weak, leaving the protocol highly susceptible to Man in the Middle (MitM) attacks.

To address these flaws, this document updates The TACACS+ Protocol to support TLS authentication and encryption, and deprecates the use of its former mechanisms.

5. Security Considerations

This document improves the integrity and authentication of the connection and network traffic between the TACACS+ servers and clients by adding TLS support. This does not in itself protect the server nor clients; the operator and equipment vendors have a role. That role is to diligently follow current best practices for maintaining the integrity of network devices and selection of TLS key and encryption algorithms.

6. IANA Considerations

IANA is requested to allocate a new well-known TCP port number fore "tacacss", described as "TACACS over TLS/SSL".
